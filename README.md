# Simple RxJava — Собственная реализация реактивной библиотеки

## Описание проекта

Реализованы базовые компоненты реактивного потока (`Observable`, `Observer`), операторы `map`, `filter`, `flatMap`, управление потоками через `Scheduler`, механизм отмены (`Disposable`).

## Архитектура

### Основные компоненты и их взаимодействие
Observable.create(source)
.map(mapper)
.filter(predicate)
.subscribeOn(scheduler)
.observeOn(scheduler)
.subscribe(observer)


**Поток данных и подписка:**

1. **Создание источника:** `Observable.create()` возвращает `ObservableCreate`, который хранит функцию-источник (`ObservableOnSubscribe`).
2. **Применение операторов:** Каждый оператор (`map`, `filter`, `flatMap`, `subscribeOn`, `observeOn`) возвращает новый `Observable`, который оборачивает предыдущий.
3. **Подписка:** При вызове `subscribe()` строится цепочка `Observer`'ов **снизу вверх**:
    - Конечный `Observer` (например, лямбда) оборачивается в `ObserveOnObserver`, если был `observeOn`.
    - Затем в `SubscribeOnObserver` (если `subscribeOn`), затем в `MapObserver` и т.д.
    - В итоге самый верхний `Observer` передаётся в метод `subscribeActual()` самого первого `Observable`.
4. **Выполнение:** `subscribeActual()` источника запускает генерацию данных. Данные проходят по цепочке операторов в обратном направлении (сверху вниз) к конечному подписчику.

**Управление потоками:**
- `subscribeOn` определяет, в каком потоке будет выполняться код **источника** (и всех операторов выше по цепочке до первого `observeOn`).
- `observeOn` переключает исполнение **нижележащих операторов и подписчика** на указанный планировщик.

### Ключевые классы

| Класс                   | Назначение                                                                                   |
|-------------------------|----------------------------------------------------------------------------------------------|
| `Observable<T>`         | Абстрактный источник данных, фабричные методы `create`, `just`, `fromCallable`, операторы.    |
| `Observer<T>`           | Интерфейс подписчика: `onSubscribe`, `onNext`, `onError`, `onComplete`.                       |
| `Disposable`            | Позволяет отменить подписку и остановить получение элементов.                                  |
| `Scheduler`             | Абстракция над пулом потоков; метод `execute(Runnable)` возвращает `Disposable` для отмены.   |
| Операторы               | Реализованы отдельными классами (`ObservableMap`, `ObservableFilter`, `ObservableFlatMap`).   |
| `*Observer` (внутренние)| Реализуют `Observer` и `Disposable`, обеспечивают преобразование и передачу событий.          |

## Schedulers (управление потоками)

Реализовано три планировщика, аналогичных RxJava:

### 1. `Schedulers.io()`
- **Пул потоков:** `CachedThreadPool` (неограниченно расширяемый).
- **Назначение:** Для задач ввода-вывода (работа с сетью, файлами, БД), где потоки могут длительное время ожидать ответа.
- **Особенности:** Потоки создаются по мере необходимости и удаляются после 60 секунд простоя.

### 2. `Schedulers.computation()`
- **Пул потоков:** `FixedThreadPool` с числом потоков = `Runtime.availableProcessors()`.
- **Назначение:** Для CPU-интенсивных вычислений (обработка данных, сложные расчёты).
- **Особенности:** Ограниченное количество потоков предотвращает деградацию производительности из-за избыточного переключения контекста.

### 3. `Schedulers.single()`
- **Пул потоков:** `SingleThreadExecutor` (один фоновый поток).
- **Назначение:** Для последовательных операций, где важен порядок обработки.
- **Особенности:** Все задачи выполняются строго одна за другой.

## Примеры использования

### Базовые операторы

```java
Observable.just(1, 2, 3, 4, 5)
    .filter(x -> x % 2 == 0)      // оставляем чётные
    .map(x -> "Even: " + x)       // преобразуем в строку
    .subscribe(System.out::println);

// Вывод:
// Even: 2
// Even: 4
Асинхронное выполнение

Observable.fromCallable(() -> {
        Thread.sleep(500);          // имитация долгой операции
        return "Результат";
    })
    .subscribeOn(Schedulers.io())   // выполнять источник в IO-пуле
    .observeOn(Schedulers.single()) // обрабатывать результат в single-потоке
    .subscribe(data -> System.out.println("Получено: " + data));
flatMap для трансформации в несколько элементов

Observable.just("A", "B")
    .flatMap(s -> Observable.just(s + "1", s + "2"))
    .subscribe(System.out::println);

// Вывод (порядок не гарантирован):
// A1
// A2
// B1
// B2
Обработка ошибок и отмена подписки

Disposable d = Observable.create(emitter -> {
        for (int i = 0; i < 100; i++) {
            if (emitter.isDisposed()) break; // проверка отмены
            emitter.onNext(i);
            Thread.sleep(10);
        }
        emitter.onComplete();
    })
    .subscribe(
        item -> System.out.println("Получено: " + item),
        err -> System.err.println("Ошибка: " + err.getMessage())
    );

Thread.sleep(150);
d.dispose(); // останавливаем генерацию
System.out.println("Подписка отменена");
